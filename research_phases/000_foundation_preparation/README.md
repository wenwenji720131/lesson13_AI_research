# Phase 0 基础准备（Foundation Preparation）

**Phase 0 的核心目标：**

> 从“对某领域感兴趣的人”，转变为“具备进入该领域科研问题探索能力的人”。

它不是正式科研阶段，而是**科研能力初始化阶段**。
这个阶段的产物不是论文，而是：

* 具备阅读和理解论文的能力；
* 掌握研究领域的基本语言；
* 能判断一个问题是否值得研究；
* 能独立进入 Phase 1（Literature Mapping）。

---

# 一、Phase 0 的主要工作内容

可以分为五个部分：

```
Phase 0 基础准备

1. 领域认知准备
2. 理论知识准备
3. 工具技能准备
4. 科研方法准备
5. 研究环境准备
```

---

# 1. 领域认知准备（Domain Orientation）

## 工作目标

回答：

> “我要研究的领域到底是什么？”

很多新人直接读论文，会遇到问题：

* 看不懂术语；
* 不知道论文重要性；
* 不知道为什么这个问题值得研究。

因此首先需要建立领域地图。

---

## 具体工作

### 1.1 确定研究领域边界

例如：

太宽：

> 人工智能

缩小：

> 深度学习

进一步：

> 医学影像中的深度学习

进一步：

> 小样本医学影像分割方法

目标：

形成：

```
大领域
 ↓
子领域
 ↓
具体问题
```

---

### 1.2 学习领域基本概念

包括：

* 核心术语
* 基础模型
* 经典方法
* 发展历史
* 主要研究机构/团队

例如机器学习：

需要知道：

* supervised learning
* unsupervised learning
* CNN
* Transformer
* diffusion model

而不是一开始就钻某篇最新论文。

---

### 1.3 找领域“地图论文”

不是马上读100篇论文。

优先：

* 教科书
* tutorial
* survey/review paper
* 经典论文

目标：

建立：

```
领域知识树

        AI
        |
     Deep Learning
        |
   Computer Vision
        |
    Medical Imaging
        |
 Image Segmentation
```

---

# 2. 理论知识准备（Theoretical Foundation）

## 工作目标

达到：

> 能够理解论文中的核心思想，而不是逐字翻译。

---

## 具体工作

根据领域不同：

### 理工科：

需要：

* 数学基础
* 模型理论
* 实验原理

例如：

AI：

* 线性代数
* 概率统计
* 优化方法
* 神经网络基础

材料：

* 物理基础
* 化学机制
* 表征方法

---

### 社会科学：

需要：

* 理论框架
* 研究范式
* 统计方法

---

## 判断标准

不是：

“学完所有基础知识。”

这是无限任务。

而是：

> 学到可以开始读领域论文。

# 4. 科研方法准备（Research Methodology）

这是很多新人缺失的部分。

目标：

理解：

> 科研不是学习知识，而是产生新知识。

---

## 需要理解：

### 什么是研究问题？

不是：

> 我想研究XX技术。

而是：

> XX技术目前存在什么限制？

---

### 什么是创新？

不是：

> 我换了一种实现方式。

而是：

> 我解决了过去方法无法解决的问题。

---

### 什么是科学证据？

包括：

* 实验验证
* 理论证明
* 数据支持
* 对比分析

---

## 资源

## 工作方式

# 二、Phase 0 的边界（Boundary）

非常重要。

Phase 0 **不是做研究**。

它的边界如下：

---

## 不应该深入：

### ❌ 不应该开始大量实验

例如：

“我先训练100个模型看看。”

问题：

你还不知道为什么做。

---

### ❌ 不应该急于提出创新点

例如：

读两篇论文：

> 我想到一个新算法。

通常是不成熟的。

---

### ❌ 不应该追求论文产出

Phase 0 不是：

“我要发论文。”

而是：

“我要具备发现论文机会的能力。”

---

# Phase 0 的结束标志

不是：

“我学了半年。”

而是：

达到以下状态：

---

## 能力边界

### 1. 能读懂论文

例如：

看到：

> We propose a novel framework based on...

能够理解：

* 输入是什么
* 方法是什么
* 输出是什么
* 为什么有效

---

### 2. 能描述领域结构

例如：

别人问：

> 这个方向现在有哪些方法？

你可以回答：

```
传统方法：
A、B

深度学习方法：
C、D

目前问题：
E、F
```

---

### 3. 能提出初步问题

不是解决方案。

例如：

不好：

> 我要设计一个Transformer模型。

好：

> 当前方法在小样本情况下性能下降，原因可能是什么？

---

# 三、Phase 0 的验证目标（Validation Goals）

Phase 0 最重要的是验证：

> “我是否已经具备进入科研探索阶段的资格？”

可以设计5个验证指标。

---

# Validation 1：文献理解能力

## 测试：

选择一篇领域经典论文。

要求：

不用看解释，回答：

1. 研究问题是什么？
2. 为什么重要？
3. 方法是什么？
4. 创新在哪里？
5. 局限是什么？

通过标准：

能够完整讲给别人听。

---

# Validation 2：领域地图能力

输出：

一张领域知识地图：

例如：

```
Research Area

├── Method A
│     ├── advantage
│     └── limitation
│
├── Method B
│
└── Open Problems
```

---

# Validation 3：复现基础能力

不要求创新。

要求：

能够：

* 跑通已有代码
* 重现实验
* 生成结果

目的：

证明你具备进入实验阶段能力。

---

# Validation 4：科研表达能力

能够写：

一页Research Summary：

包含：

```
研究领域：
已有工作：
主要问题：
可能方向：
```

---

# Validation 5：提出问题能力

给一个论文：

能够提出：

至少3个：

* limitation
* improvement direction
* open question

---

# 四、Phase 0 的最终交付物（Deliverables）

建议形成：

| 产物     | 说明          |
| ------ | ----------- |
| 领域地图   | 研究方向结构      |
| 核心论文列表 | 20~50篇      |
| 术语表    | 领域语言        |
| 论文阅读笔记 | 理解证明        |
| 工具环境   | 实验条件        |
| 复现案例   | 能力证明        |
| 研究兴趣说明 | 进入Phase 1基础 |

---

# 五、Phase 0 与 Phase 1 的分界

一句话：

**Phase 0：学习如何看问题。**

**Phase 1：开始寻找值得解决的问题。**

区别：

|    | Phase 0 | Phase 1  |
| -- | ------- | -------- |
| 问题 | 别人研究什么？ | 还有什么没解决？ |
| 目标 | 理解领域    | 发现机会     |
| 阅读 | 经典+综述   | 大量最新论文   |
| 输出 | 知识地图    | 研究gap    |
| 身份 | 学生      | 初级研究者    |

---

因此，一个完整科研流程中：

> Phase 0 不是“准备阶段”这么简单，它实际上是在建立科研操作系统。

没有 Phase 0，后面的 Literature Review 很容易变成“论文搬运”；
有了 Phase 0，Literature Review 才会逐渐变成“寻找未知问题”。
