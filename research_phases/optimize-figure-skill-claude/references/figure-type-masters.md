# 图类型融合规则

本文件不是可直接复制的长提示词母版，而是 Skill 内部使用的图类型适配层。不要逐段拼接本文件与领域母版。

融合原则：
- 领域母版负责科学对象、学科语义、期刊/会议视觉语言和 Stage 2 渲染方式。
- 图类型规则只约束表达目标、布局候选、区域组织、连接关系、可见文字和常见失败。
- 如果领域母版和图类型规则冲突，优先级为：科学事实 > 用户明确要求 > 领域母版 > 图类型结构 > 模型适配 > 审美词。
- 最终 prompt 必须包含可见文字限制和科学边界，不得让绘图模型编造数据、设备、机制或实验结果。

内部 Figure Brief 建议字段：
1. 图像目标：一句话说明这张图帮助读者理解什么。
2. 领域对象：来自领域母版的具体视觉实体和学科术语。
3. 图类型结构：来自本文件的布局、区域和连接规则。
4. 可见文字：图中允许出现的标签清单。
5. 科学边界：不能新增、不能编造、不能误画的内容。
6. 最终绘图 prompt：融合领域母版、图类型规则、模型适配和语言策略后的单段可复制 prompt。

Figure Brief 是本 Skill 多轮迭代的锚点：第一轮生成后完整保留 Figure Brief 的 6 个字段（尤其是区域/连接关系/可见文字清单），后续每一轮迭代都基于同一份 Figure Brief 做最小修订，不重新推断，除非用户的反馈明确要求改变图类型或领域判断本身。

## 1. 技术路线图 / 研究流程图

目的：展示研究步骤、方法路径和输出关系。

适合内容：Methods、研究方案、开题技术路线、模型/实验流程、数据处理流程。

布局候选：
- 横向主流程：适合线性方法步骤。
- 纵向流程：适合阶段递进或报告页面。
- 三段式：问题/输入 -> 方法/分析 -> 输出/验证。
- 分层流程：适合主流程下包含若干子模块。

必须抽取：
- 研究起点或问题背景。
- 3-6 个关键步骤。
- 最终输出、验证对象或结论指向。
- 箭头方向、分支、反馈或并列关系。

可见文字：
- 步骤名、模块名、输入/输出短标签。
- 不要生成长解释、随机图例、未列出的标题或脚注。

常见失败：
- 画成普通装饰流程框，缺少学科对象。
- 新增论文没有说明的步骤。
- 把方法流程误画成图形摘要或机制图。
- 箭头方向与研究流程相反。

融合规则：
- 领域母版决定步骤内部的学科对象怎么画。
- 本类型只约束"步骤推进"和"路径清晰"。

## 2. 实验系统 / 装置结构示意图

目的：展示实验系统由哪些真实部件组成，以及它们如何连接和协同工作。

适合内容：实验平台、仪器结构、反应器/传感器/采集系统、测试流程中的设备关系。

布局候选：
- 中心装置 + 周围功能模块。
- 输入端 -> 中心反应/测试区 -> 输出/采集端。
- 上下分区：实验对象 / 测量系统 / 数据分析。

必须抽取：
- 实验对象或测试对象。
- 仪器、装置、传感器、反应器、采集模块等真实出现的部件。
- 物料流、气流、电信号、数据流或样品转移关系。
- 必须保留和必须避免的标注。

可见文字：
- 设备名、模块名、输入/输出短标签。
- 必须来自论文内容或用户说明。

常见失败：
- 虚构设备、传感器、管路、参数或制造细节。
- 画成产品广告、写实照片、重金属反光或复杂机械剖面。
- 连接关系过密，像线路板或工程施工图。

融合规则：
- 领域母版决定装置中的材料、样品、生物实体或计算模块的准确表达。
- 本类型只约束"真实部件 + 连接关系 + 简化学术示意图"。

## 3. 机制解释图

目的：解释现象、反应、调控或作用路径如何发生。

适合内容：作用机制、反应路径、调控通路、材料演化、病理/药理机制、算法内部作用关系。

布局候选：
- 因果链：A 导致 B，B 进一步导致 C。
- 中心对象向外展开：核心实体连接多个作用因素。
- 多尺度放大：宏观对象 -> 微观结构 -> 分子/模块。
- 对照机制：处理前后、正常/异常、方法 A/B。

必须抽取：
- 被解释的核心现象或过程。
- 关键实体和作用环境。
- 促进、抑制、迁移、转化、传递、反馈等关系。
- 论文已经支持的机制边界。

可见文字：
- 关键实体名、过程名、作用关系短标签。
- 文字要少而准确。

常见失败：
- 编造中间机制、公式、参数或结果。
- 把机制图画成研究流程图。
- 用夸张特效代替科学关系。
- 箭头语义不清，促进/抑制/转化混在一起。

融合规则：
- 领域母版决定实体形态，例如晶格、离子、细胞、蛋白、神经网络模块或系统组件。
- 本类型只约束"机制主线 + 箭头语义 + 科学边界"。

## 4. 多面板比较图

目的：比较不同组别、条件、方法、材料或阶段之间的差异。

适合内容：结果比较、工况比较、消融实验、材料/样本对照、治疗组与对照组。

布局候选：
- 2x2 面板。
- 横向三联或四联。
- 上下双排。
- 左侧条件说明 + 右侧统一对比面板。

必须抽取：
- 比较对象和组别名。
- 每个 panel 展示什么。
- 哪些是真实数据，哪些只能定性示意。
- 统一尺度、视角、配色、图例和边距规则。

可见文字：
- panel 编号、组别名、必要标签。
- 不要新增坐标、色标、显著性标记或数值，除非原文提供。

常见失败：
- 每个 panel 风格不同，失去可比性。
- 伪造数据曲线、显著性、热图色标或坐标刻度。
- 把比较图画成图形摘要或宣传拼图。

融合规则：
- 领域母版决定每个 panel 中的学科对象或图形语言。
- 本类型只约束"统一比较结构"和"数据边界"。

## 5. 图形摘要 / 论文主图

目的：把论文的研究问题、核心方法、关键过程和主要贡献压缩成一张主图。

适合内容：Graphical Abstract、论文主图、投稿摘要图、一图讲清贡献。

布局候选：
- 问题 -> 方法 -> 发现/贡献。
- 左中右叙事。
- 中心发现向外展开。
- 3-4 个区域的多区域摘要。

必须抽取：
- 研究问题或痛点。
- 核心方法、材料、模型或系统。
- 关键过程、机制或验证路径。
- 主要发现、贡献或应用价值。

可见文字：
- 少量短标签，突出主线。
- 避免长句、段落和过密术语。

常见失败：
- 把全文所有细节塞进一张图。
- 变成普通流程图或单一机制图。
- 过度宣传化，弱化科学对象。
- 夸大贡献或虚构结果。

融合规则：
- 领域母版决定主视觉对象和学科气质。
- 本类型只约束"一图讲清论文贡献"的叙事压缩。

## 6. 开题 / 答辩 / 汇报总览图

目的：概括课题整体结构、研究模块和汇报主线。

适合内容：开题首页、答辩总览页、组会汇报首页、项目研究框架。

布局候选：
- 中心主题 + 周围研究模块。
- 上中下：背景/问题 -> 研究模块 -> 预期输出。
- 左右推进：问题 -> 路径 -> 结果。
- 模块矩阵：多个研究内容并列展示。

必须抽取：
- 研究背景和核心问题。
- 研究目标。
- 2-4 个主要研究模块。
- 技术路径、阶段安排或成果输出。

可见文字：
- 汇报首页可读的短标题。
- 不要写密集说明段落。

常见失败：
- 画成商业路演、咨询公司框架图或装饰封面。
- 把计划画成已完成结论。
- 模块层级混乱，听众无法快速抓住课题结构。

融合规则：
- 领域母版决定各模块中的学科对象。
- 本类型只约束"总览层级"和"演示可读性"。

## 7. 期刊封面图 / Cover Art

目的：把研究亮点转化为视觉隐喻和封面主视觉。

适合内容：期刊封面、内封面、论文宣传图、视觉 teaser。

布局候选：
- 竖版封面构图，一个强中心主视觉。
- 中心科学对象 + 象征性环境。
- 宏观场景与微观机制融合。
- 抽象视觉隐喻，但保留科学对象可识别性。

必须抽取：
- 最适合封面化表达的研究对象。
- 核心机制、结构、生命过程、算法概念或应用场景。
- 可转化为视觉隐喻的关键词。
- 不能艺术化误读的科学边界。

可见文字：
- 默认不生成任何文字。
- 不得生成期刊 Logo、刊名、卷期号、条形码、作者名或真实出版物元素。

常见失败：
- 变成广告、奇幻场景、产品海报或仿真期刊封面。
- 艺术化过度导致科学对象不可识别。
- 把隐喻画成实验结果。

融合规则：
- 封面图不走普通白底论文图 Stage 2。
- 使用专门的封面渲染规则：竖版、强主视觉、高审美，但科学对象和机制不能失真。
