# 自适应领域母版与参考图风格

仅在用户需求超出现有领域母版，或用户提供参考图/目标风格时使用本参考文件。这里区分两个模式：

1. **未知领域自适应母版**：为 `domain-masters.md` 尚未覆盖的学科生成可复用领域母版。
2. **参考图风格补丁**：从参考图提取可迁移风格，附加到已有母版后，不复制参考图的科学内容。

这两个模式都发生在**第一轮 prompt 生成之前或期间**（用户提供的是论文相关的参考图/期望风格图）。它们和 `iteration-critique.md` 描述的"多轮迭代优化"是两条不同路径，不要混淆：

- 本文件的参考图 = 用户希望"最终图长得像这张图"的**风格来源**，用于影响第一版 prompt 的生成。
- `iteration-critique.md` 的反馈图 = 用户拿到的**上一轮 prompt 生成的结果图**，用于诊断问题、修订下一版 prompt。

如果用户在已经进入多轮迭代后才提供一张新的风格参考图，先判断这张图是"新的风格灵感"还是"上一版 prompt 的生成结果"；如果不确定，直接询问用户。

## 模式 A：未知领域自适应母版

当用户研究领域无法稳定匹配 CS/ML、材料/化学、生物/医学三类内置领域母版时使用，尤其适合环境科学、地质、能源系统、土木、农业、生态、心理学、教育学、公共卫生、遥感、政策建模等交叉或专门领域。

如果用户同时提供参考图，且当前模型具备读图能力，可以把参考图作为视觉参考。若当前模型不能读图，应要求用户补充参考图的布局、配色、图标、标签和整体风格描述，再生成新母版。

### 来源逻辑

本模式融合：

- 定制化元提示词思路：用顶级发表图推断领域专属排版和视觉语义。
- 新领域母版格式：产物不是短风格描述，而是一套可保存、可复用的完整领域母版。

### 需要推断的内容

只推断可复用的领域级规律：

- 常见研究对象和可视化实体。
- 常见过程逻辑、机制链或因果链。
- 该领域常见图示类型。
- 布局模式和信息层级。
- 领域合适的视觉语言、符号、纹理、尺度和隐喻。
- 字体、标签密度和注释习惯。
- 准确性、伦理、数据和实证边界。

不要复制：

- 参考图中的具体科学结论。
- 数据值、曲线、地图、显微/照片证据、测量值、标签、样本名或实验结果。
- logo、水印、专有模板或具有明显版权风险的独特构图细节。

### 新母版输出格式

```markdown
## 新领域判断
[领域名称；为什么现有母版不够；是否使用参考图；从参考图推断出的可复用领域视觉线索。]

## 新领域母版草案

# [领域名称] 领域母版

## 适合场景
[这个母版覆盖哪些图类型和研究语境。]

## 领域图示特征
[对象、布局、层级、配色、符号、箭头、尺度线索、字体和标签密度。]

## Stage 1: Visual Schema
[完整 schema 生成提示词，包含角色、目标、布局选择、领域语义、视觉对象规则、连接规则和输出格式。]

## Stage 2: Rendering Prompt
[完整渲染提示词，包含艺术风格、布局执行、视觉规则、{VISIBLE_TEXT_RULE} 和禁止内容约束。]

## 可替换变量
[研究主题、对象列表、阶段、标签、图类型、目标期刊/会议、模型、语言策略。]

## 稳定约束
[不建议删除的约束，用于维持科学准确性和视觉一致性。]

## 质检清单
[8-10 项检查：科学结构、领域语义、视觉一致性、标签、实证边界、目标期刊/会议要求等。]

## 本次最终绘图提示词
[用新母版草案为用户当前任务生成一段可直接复制的 prompt。]

## 是否保存为 Skill 母版
如果用户满意，询问是否将该领域母版保存到 `references/domain-masters.md`，以后同领域请求可直接调用。
```

### 保存协议

只有在用户明确同意后才保存新领域母版。保存时：

- 在 `references/domain-masters.md` 末尾追加新的 `## [领域名称]` 小节。
- 保持与现有领域母版一致的两阶段结构。
- Stage 2 中保留 `{VISIBLE_TEXT_RULE}` 占位符。
- 如果该领域可能复用，增加或更新 eval fixture。
- 如果用户确认保存新母版，只更新当前 Skill 的 `references/domain-masters.md`。

## 模式 B：参考图风格补丁

当已有领域母版或图类型母版适用，但用户希望输出接近某张参考图的风格时使用。

默认产物是风格补丁，而不是新母版。

### 读图能力判断

- 如果当前模型能查看上传图片，直接分析图片。
- 如果当前模型不能读图，要求用户描述参考图，或切换到具备读图能力的模型。
- 不要假装看过不可用或无法读取的图片。

### 分析维度

提取可迁移设计属性：

1. 布局与构图：方向、阶段、面板结构、视觉中心、留白。
2. 信息层级：主次模块、分组方式、强调方式。
3. 容器与形状：矩形、圆角面板、有机形态、晶格、图标、标注框。
4. 线条与箭头：粗细、曲直、虚实语义、箭头样式。
5. 配色：色相、饱和度、对比度、背景、分组颜色。
6. 字体与标签：字体风格、标签密度、位置、大小写。
7. 纹理与深度：扁平矢量、轻微阴影、等距透视、纸张/界面/材料质感。
8. 领域语义：仅提取与风格有关的部分，不复制参考图科学内容。

### 输出格式

```markdown
## 参考图风格摘要
[5-8 条简洁总结可迁移风格。]

## 可添加到最终 prompt 的风格补充
[一段可附加到所选母版后的风格说明，覆盖风格、布局、配色、箭头、字体和避免事项。]

## 不应复制的内容
[不能迁移的内容类型：标签、数据、模块、结果结论、logo、水印、精确构图等。]

## 适用边界
[说明该风格适合和不适合哪些图类型。]

## 最终绘图提示词
[所选母版 + 用户研究内容 + 风格补丁 + 可见文字规则。]
```

### 集成规则

风格补丁不能覆盖科学结构、可见文字限制、实证边界或模型相关标签密度规则。所选母版仍然是主体，参考图风格补丁只作为附加层。
